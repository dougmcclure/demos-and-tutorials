Event Analysis App Walk Through


Step 1- Launch 1-EventHotSpots-EventTrendBySeverityA common starting point for event analysis includes analyzing the event volume trends over time. In this app, we're showing five very common dimensions for analyzing the event volumne trends by severity.
  • The first chart shows Event Trends by Severity for the past hour aggregated at the minute level.
  • The second chart shows Event Trends by Severity for the past day aggregated at the hour level.
  • The third chart shows Event Trends by Severity for the past week aggregated at the day level.
  • The fourth chart shows Event Trends by Severity for the past month aggregated at the week level.
  • The fifth chart shows Event Trends by Severity for the past year aggregated at the month level.
Step 2- Launch 2-EventHotspots-NodeThis event analysis app includes analysing events originating by the source end point system, EMS/NMS, probe/integration gateway, etc. This could also be modified to look at the manager field to see the top event volumes by source system or integration.
  • The first chart shows Top 20 Nodes with the highest event counts over the past day.
  • The second chart shows Top 20 Nodes with the highest event counts over the past week.
  • The third chart shows Top 20 Nodes with the highest event counts over the past month.
  • The fourth chart shows Top 20 Nodes with the highest event counts over the past year.
Step 3- Launch 3-EventHotSpots-AlertGroupThis event analysis app includes analysing events originating by the type of monitoring agent, situation, or similar classification commonly captured in the AlertGroup field.

  • The third chart shows Top 20 AlertGroups with the highest event counts over the past month.
